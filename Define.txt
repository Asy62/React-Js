// Real DOM vs Virtual DOM //


Real DOM : Real dom is the actual tree-like structure of HTML elements created by the browser for a web page.
           It directly represent the UI on the screen
           Any change in the Real DOM updates the UI immediately.
           Updating it is slow, because the browser has to re-render the page.


Virtural DOM : Virtual DOM is a lightweight JavaScript copy of the Real DOM used by libraries like React. 
               It does not update the UI directly.
               Changes are first made in the Virtual DOM.
               Then only hte changed parts are updated in the Real DOM.
               This makes updates faster and more efficient.

// Hooks//
Hooks are special functions in React that let you use state, lifecycle features, and other React capabilities inside functional components- no class components needed.


1.Write cleaner code, & shorter code.
2. Reuse logic easily (via custom hooks)
3. Avoid complex class syntax (this, constructors, binding)


// Commonly Used React Hooks //
1. useState - State management---> Used to store and update data in a component.
2. useEffect - Side effects ----> used for things like 1.API Calls,
                                                       2. Timers
                                                       3. DOM updates
                                                       4. Subscriptions
3. useContext - Global data sharing ---> Avoids prop drilling.
4. useRef - Direct DOM access---> Used to access DOM elements or persist values without re-render.
5. useMemo - Performance optimization ----> Prevents expensive recalculations.
6. useCallback -Memoize function --> Prevents function recreation on re-renders.
7. Custom Hooks ----> Create your own hooks to reuse logic.
    (custom hooks must start with use)


// Rule of Hooks //
1. Call hooks only at the top level.
2. Call hooks only inside React functions.
3. Don't use hooks inside loops or conditions.
                                                    



                              // Local Storage //

Local Storage lets you store data in the browser even after page refresh or browser restart.

                // useEffect Hook //
useEffect is a React Hook used to handle side effects in functional components after rendering.


What is Side Effects?
side effects = operations that affect somethings outside React rendering.
Examples: 
* API calls,
* Local Storage.
* Timers
* DOM updates
* Event listeners.


1. Run Only Once (On Mount)

useEffect(() => {
  console.log("Component Mounted");
}, []);

Empty array = runs only once

📌 Use case:

API call

Initial setup

2. Run on Every Render

useEffect(() => {
  console.log("Runs every render");
});

No dependency array.

✅ 3. Run When Dependency Changes
useEffect(() => {
  console.log("Runs when count changes");
}, [count]);

Runs only when count changes.


What is Axios?
Axios is a promise-based HTTP Client for node.js and the browser. It is isomorphic (= it can run in the browser and node.js with the same codebase). On the server-side it uses the native node.js http module, while on the client (browser) it uses XMLHttpRequest.

Features
Make XMLHttpRequest from the browser
Make http requests from node.js
Supports the Promise API
Intercept request and response
Transform request and response data
Cancel requests

Installing :- npm install axios
